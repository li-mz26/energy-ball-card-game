# 数风流 - 大战略模式设计文档

## 1. 游戏模式概述

**模式名称**: 天下统一战  
**核心玩法**: 地图战略 + 卡牌对战的双层结构

### 1.1 游戏流程

```
[选择出生点] → [郡县争霸] → [区域统一] → [轮回继承] → [再次争霸] → ... → [最终失败]
```

### 1.2 双层结构

| 层级 | 内容 | 操作 |
|------|------|------|
| **战略层** | 地图、郡县、势力移动 | 选择进攻目标、调兵遣将 |
| **战术层** | 卡牌对战 | 布阵、输送、战斗 |

---

## 2. 地图系统

### 2.1 地图结构

采用**节点图**结构，每个节点代表一个郡/州：

```
        [幽州] —— [并州]
           |         |
        [冀州] —— [青州] —— [徐州]
           |         |         |
        [司隶] —— [兖州] —— [豫州] —— [扬州]
           |         |         |         |
        [凉州] —— [关中] —— [荆州] —— [江东]
           |                   |
        [益州] —— [汉中] —— [交州]
```

### 2.2 郡县属性

每个郡县包含：
- **名称**: 郡县名（如"颍川"、"河东"）
- **所属州**: 所属的大州（如"豫州"）
- **相邻郡县**: 可进攻的目标
- **当前势力**: 占领者
- **特产**: 影响将领招募（如"颍川出谋士"）
- **地形**: 影响防守加成

### 2.3 势力范围

- 每个势力控制一个或多个相邻郡县
- 势力颜色区分
- 势力名称可自定义或随机（如"刘氏集团"、"曹氏集团"）

---

## 3. 势力系统

### 3.1 势力属性

| 属性 | 说明 |
|------|------|
| 名称 | 势力名（如"蜀汉"、"曹魏"） |
| 君主 | 势力的代表将领 |
| 控制郡县 | 当前拥有的郡县列表 |
| 将领池 | 可使用的将领（最多11人上场） |
| 势力色 | 地图显示颜色 |
| 传承等级 | 轮回次数，影响初始优势 |

### 3.2 玩家势力

- 初始选择一个郡县作为根据地
- 初始获得5名随机将领
- 每占领一个新郡县，获得该郡县特产将领

### 3.3 AI势力

- 随机分布在地图上
- 每回合自动扩张
- 势力强度随轮回次数增加

## 4. 战略回合循环

### 4.1 回合流程

```
┌─────────────────────────────────────────────────────────────┐
│                      战略回合开始                            │
├─────────────────────────────────────────────────────────────┤
│  1. 势力行动阶段                                            │
│     - 玩家选择进攻目标（相邻郡县）                          │
│     - AI势力自动选择进攻/防守                               │
│     - 多个势力冲突时，随机两两配对                          │
├─────────────────────────────────────────────────────────────┤
│  2. 战斗判定阶段                                            │
│     - 每一对交战势力进入战术对战                            │
│     - 使用卡牌对战规则决出胜负                              │
├─────────────────────────────────────────────────────────────┤
│  3. 领土变更阶段                                            │
│     - 胜者吞并败者的郡县                                    │
│     - 败者若失去所有郡县，势力灭亡                          │
│     - 灭亡势力的将领进入流浪池                              │
├─────────────────────────────────────────────────────────────┤
│  4. 检查统一                                                │
│     - 若只剩一个势力，本轮结束，进入轮回                    │
│     - 否则进入下一回合                                      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 战斗触发规则

- 玩家可选择任意相邻郡县进攻
- AI势力优先进攻弱小的相邻势力
- 三方冲突时，最强两方先战，胜者再与第三方战

### 4.3 郡县特产与招募

| 郡县类型 | 特产倾向 | 招募加成 |
|---------|---------|---------|
| 颍川 | 谋士型将领 | 调度+5 |
| 河东 | 武将型将领 | 武勇+5 |
| 幽州 | 骑兵将领 | 截击范围+1 |
| 江南 | 水军将领 | 输送成功率+5% |
| 益州 | 防守型将领 | 接应+5 |

## 5. 轮回继承系统

### 5.1 轮回触发

- 当地图被统一（只剩一个势力）时，本轮结束
- 若玩家是统一者，进入下一轮回
- 若玩家被灭亡，游戏结束

### 5.2 继承规则

**保留内容**:
- 所有已招募的将领
- 将领的等级、装备
- 已解锁的郡县图鉴

**重置内容**:
- 地图重新分割（随机分配）
- 所有势力回到初始状态
- 初始郡县重新选择

### 5.3 难度递增

每轮回增加难度：
- AI势力初始将领数+1
- AI势力初始等级+2
- 新郡县特产将领品质降低（橙→紫→蓝）

### 5.4 轮回奖励

根据统一时的表现：
- **快速统一**（10回合内）: 获得1名随机橙将
- **全胜统一**（未战败）: 所有将领经验+1000
- **以少胜多**（初始郡县≤3）: 解锁特殊郡县

---

## 6. 将领获取与养成

### 6.1 初始将领

- 选择出生郡县时，获得该郡县历史名将3名
- 额外随机获得2名将领

### 6.2 征服招募

- 占领新郡县时，可招募该郡县1名特产将领
- 选择范围：该郡县历史上的真实人物

### 6.3 流浪招募

- 灭亡势力的将领进入流浪池
- 每回合可用资源招募1名流浪将领

### 6.4 上限控制

- 将领池上限：30人
- 超出上限时需解雇或融合

---

## 7. 胜利与失败

### 7.1 胜利条件

- 完成第N轮回的统一（N可设置，如5轮回）
- 最终统计：总统一次数、总将领数、总胜率

### 7.2 失败条件

- 玩家势力被灭亡（失去所有郡县）
- 可观看AI继续争夺，或重新开始

### 7.3 排行榜

- 最多轮回数
- 最快统一记录
- 最多将领收集

## 9. 技术实现

### 9.1 新增模块

```
src/
├── strategy/
│   ├── map.lua           # 地图数据与节点图
│   ├── county.lua        # 郡县类
│   ├── faction.lua       # 势力类
│   └── campaign.lua      # 战役管理器
├── data/
│   └── county_data.lua   # 郡县数据（特产、相邻关系）
└── ui/
    └── strategy_ui.lua   # 战略地图UI
```

### 9.2 数据结构

**地图节点图**:
```lua
MapGraph = {
    counties = {},      -- 所有郡县
    adjacency = {},     -- 邻接表
    factions = {},      -- 所有势力
    currentFaction = 1  -- 当前行动势力
}
```

**战役记录**:
```lua
Campaign = {
    round = 1,          -- 当前轮回
    turn = 1,           -- 当前回合
    battles = {},       -- 本回合待进行的战斗
    unified = false     -- 是否已统一
}
```
