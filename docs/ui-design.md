# 能量球 (Energy Ball) - UI设计文档

## 1. 设计原则

### 1.1 视觉风格

- **主题**: 科幻能量 / 赛博朋克
- **色调**: 深蓝(#1a1a2e)为主，青色(#00d4ff)和紫色(#9d4edd)为强调色
- **氛围**: 紧张、策略、科技感

### 1.2 设计目标

1. 清晰展示战场状态和能量球传递路径
2. 直观呈现单位信息和能力
3. 简洁高效的布阵操作
4. 沉浸式的战斗反馈

## 3. 界面详细设计

### 3.1 顶部状态栏

```
┌─────────────────────────────────────────────────────────────────────┐
│  [Logo]    第2轮 / 第3轮    ❤️ 45  vs  ❤️ 72    [设置] [菜单]    │
└─────────────────────────────────────────────────────────────────────┘
```

**元素说明**:
| 元素 | 尺寸 | 说明 |
|------|------|------|
| Logo | 40×40 | 游戏图标 |
| 回合信息 | 200×40 | 当前回合/总回合 |
| 生命值 | 150×40 | 双方大本营血量，带进度条 |
| 设置按钮 | 40×40 | 齿轮图标 |
| 菜单按钮 | 40×40 | 汉堡菜单图标 |

**生命值显示**:
- 使用心形图标 + 数字
- 低血量(<30%)时图标变红并闪烁
- 受伤时有血条减少动画

### 3.2 主战场区域

```

    │   └─────┘  └─────┘  └─────┘  └─────┘ │
    │      ↓        ↓        ↓        ↓    │
    │   ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐ │

    │   │     │  │ ○○○ │  │ ○○○ │  │ ○○○ │ │
```

#### 3.2.2 单位槽位设计

**单槽尺寸**: 120×160 px

```
┌─────────────────┐
│  [单位名称]     │  ← 16px 字体
├─────────────────┤
│                 │
│   [单位立绘]    │  ← 80×80 图像区
│                 │
├─────────────────┤
│ ❤️ 80  🎯 65   │  ← 生命/发送能力
│ 🛡️ 70  👁️ 5    │  ← 接收/感知范围
└─────────────────┘
```

**状态指示**:
- 可行动: 绿色边框脉冲动画
- 选中: 金色边框 + 发光效果
- 受伤: 红色闪烁
- 被击败: 灰色半透明

#### 3.2.3 能量球传递可视化

**传递路径动画**:

```
能量球从A1传递到A2:

A1 [🔵] ──────→ [🔵] A2
     ╲    /
      ╲  /   粒子轨迹
       ╲/
```

**动画效果**:
- 能量球: 发光球体，带拖尾粒子
- 路径: 虚线箭头，流动效果
- 成功: 绿色波纹扩散
- 失误: 红色X标记 + 消散效果
- 拦截: 黄色闪电效果
- 转移: 紫色传送门效果

#### 3.3.2 拖放交互

**拖动单位**:
1. 从手牌区拖拽单位卡
2. 战场槽位高亮显示可放置区域
3. 拖放到目标槽位，播放放置动画
4. 槽位已满时显示红色禁止标志

**右键菜单**:
- 查看单位详情
- 快速替换
- 返回手牌

## 4. 视觉元素规范

### 4.1 色彩系统

**主色调**:
```css
--bg-primary: #1a1a2e      /* 深蓝背景 */
--bg-secondary: #16213e    /* 次级背景 */
--bg-card: #0f3460         /* 卡片背景 */
--accent-cyan: #00d4ff     /* 青色强调 */
--accent-purple: #9d4edd   /* 紫色强调 */
--accent-gold: #ffd700     /* 金色(选中) */
--danger: #e94560          /* 红色(危险) */
--success: #00ff88         /* 绿色(成功) */
--warning: #ffb800         /* 黄色(警告) */
--text-primary: #ffffff    /* 主文字 */
--text-secondary: #a0a0a0  /* 次级文字 */
```

**阵营色**:
- 我方: 青色渐变 (#00d4ff → #0099cc)
- 敌方: 红色渐变 (#e94560 → #b8364a)
- 中性: 灰色 (#808080)

### 4.2 字体规范

**中文字体**:
- 标题: 思源黑体 Bold / 微软雅黑 Bold
- 正文: 思源黑体 Regular / 微软雅黑
- 数字: DIN / Roboto Mono

**字号**:
| 用途 | 大小 | 字重 |
|------|------|------|
| 大标题 | 32px | Bold |
| 小标题 | 24px | Bold |
| 正文 | 16px | Regular |
| 辅助文字 | 12px | Regular |
| 数值 | 20px | Bold |

### 4.3 图标系统

**属性图标** (24×24):
- ❤️ 生命值
- 🎯 发送能力
- 🛡️ 接收能力
- ⚡ 拦截能力
- 👁️ 感知范围

**功能图标** (32×32):
- 设置 (齿轮)
- 菜单 (汉堡)
- 确认 (对勾)
- 取消 (叉号)
- 信息 (感叹号)

## 5. 动画规范

### 5.1 过渡时间

| 动画类型 | 时长 | 缓动函数 |
|---------|------|---------|
| 界面切换 | 300ms | ease-out |
| 单位选中 | 200ms | ease-in-out |
| 能量球飞行 | 500ms | linear |
| 伤害数字 | 1000ms | ease-out |
| 脉冲效果 | 1500ms | ease-in-out (循环) |

### 5.2 特效规范

**发光效果**:
```css
/* 选中发光 */
box-shadow: 0 0 20px var(--accent-cyan),
            0 0 40px var(--accent-cyan);

/* 危险发光 */
box-shadow: 0 0 20px var(--danger),
            0 0 40px var(--danger);
```

**粒子效果**:
- 能量球拖尾: 5-10个粒子，随机大小
- 击中特效: 爆炸式扩散，8-12个粒子
- 消散效果: 向上飘散，逐渐透明

## 6. 响应式设计

### 6.1 桌面端 (≥1920×1080)

- 完整4列战场布局
- 手牌区横向排列
- 详细信息面板常驻

### 6.2 平板端 (1280×720 ~ 1920×1080)

- 战场布局保持4列
- 手牌区可折叠
- 详细信息面板点击弹出

### 6.3 移动端 (<1280×720)

- 战场布局保持4列但缩小
- 手牌区改为底部抽屉
- 简化部分视觉效果

## 7. 音效设计 (预留)

### 7.1 UI音效

- 按钮点击: 清脆的"滴"声
- 选中单位: 柔和的"嗡"声
- 放置单位: "咔嗒"确认声
- 错误操作: 低沉的"咚"声

### 7.2 战斗音效

- 能量球发射: "咻"的飞射声
- 传递成功: 清脆的"叮"
- 发送失误: "嘶"的消散声
- 拦截成功: "啪"的撞击声
- 造成伤害: "轰"的爆炸声
