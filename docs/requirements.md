# 能量球 (Energy Ball) - 需求文档

## 1. 游戏概述

**游戏名称**: 能量球 / Energy Ball  
**类型**: 回合制策略卡牌游戏  
**目标平台**: PC / 移动端  
**开发语言**: Lua (使用 Love2D 框架)  
**玩家人数**: 2人对抗 (支持本地对战/在线对战)

---

## 2. 核心玩法

### 2.1 战场布局

双方各拥有 **11个单位**，分为 **4排** 放置在战场上：

```
        [敌方大本营]
             ↑
    ┌────┬────┬────┬────┐
    │ A1 │ A2 │ A3 │ A4 │  ← 我方 (玩家A)
    ├────┼────┼────┼────┤
    │ B4 │ B3 │ B2 │ B1 │  ← 敌方 (玩家B)
    └────┴────┴────┴────┘
             ↓
        [我方大本营]
```

**交叉排列规则**:  
战场实际排列为 `A1 - B4 - A2 - B3 - A3 - B2 - A4 - B1` 的交替阵型。

- **第1排**: A1 (1单位) vs B4 (1单位)
- **第2排**: A2 (3单位) vs B3 (3单位)  
- **第3排**: A3 (3单位) vs B2 (3单位)
- **第4排**: A4 (3单位) vs B1 (3单位)

### 2.2 能量球传递机制

#### 2.2.1 进攻流程

当A方进攻时，能量球从 **A1** 开始，沿以下路径传递：

```
A1 → A2 → A3 → A4 → 敌方大本营(假想A5)
```

每个能量球在传递过程中会经历以下判断：

#### 2.2.2 能量球状态变化

对于每一次发送，能量球可能经历：

1. **成功发送**: 能量球到达目标排
2. **发送失误**: 能量球在传递中丢失
3. **被拦截**: 被路径上的敌方单位拦截
4. **同排转移**: 转移到同排的另一单位，由该单位继续发送

**同排转移限制**: 已被同排转移过的能量球 **不会再次被转移**。

### 2.3 发送判定详解

#### 2.3.1 目标选择

发送单位根据以下因素决定目标：
- 可观察到的目标单位状态
- 概率分布由发送单位AI/玩家策略决定

#### 2.3.2 发送失误概率

```
失误概率 = f(发送方能力值, 接收方能力值, 双方距离)
```

影响因素：
- 发送方能力值越高，失误率越低
- 接收方能力值越高，失误率越低
- 双方距离越远，失误率越高

#### 2.3.3 拦截机制

当能量球从排X传递到排Y时：
1. 系统计算传递路径
2. 找出距离路径 **最近** 的敌方单位
3. 该敌方单位执行拦截判定

```
拦截成功率 = f(拦截者属性, 拦截者到路径的距离)
```

- 拦截者属性越高，拦截成功率越高
- 拦截者距离路径越近，拦截成功率越高

### 2.4 游戏回合

#### 2.4.1 回合结构

游戏共进行 **3轮**，每轮包含：

1. **布阵阶段** (进攻方)
   - 进攻方可以重新调整每排的单位配置
   - 将手牌中的单位放置到4排中
   - 每排单位数限制：第1排1个，第2-4排各3个

2. **进攻阶段** (进攻方)
   - 从A1发射能量球
   - 能量球沿链条传递
   - 最终伤害结算

3. **防守阶段** (防守方)
   - 被动承受攻击
   - 拦截机会

#### 2.4.2 回合交替

```
第1轮: A进攻 → B防守 → B进攻 → A防守
第2轮: A进攻 → B防守 → B进攻 → A防守  
第3轮: A进攻 → B防守 → B进攻 → A防守
```

### 2.5 胜利条件

- **即时胜利**: 敌方大本营生命值归零
- **最终胜利**: 3轮结束后，生命值较高的一方获胜

## 3. 单位系统

### 3.1 单位属性

每个单位拥有以下基础属性：

| 属性 | 说明 |
|------|------|
| 名称 | 单位名称 |
| 生命值 | 单位的耐久度 |
| 发送能力 | 影响发送成功率和失误概率 |
| 接收能力 | 影响被发送时的失误概率 |
| 拦截能力 | 影响拦截成功率 |
| 感知范围 | 可观察到的目标范围 |
| 特殊技能 | 单位的独特能力 |

### 3.2 单位类型示例

**类型1: 传令兵 (Courier)**
- 高发送能力，适合作为能量球传递的起点
- 技能: "紧急传递" - 首次发送必定成功

**类型2: 接收员 (Receiver)**  
- 高接收能力，减少接收时的失误
- 技能: "稳定接收" - 被发送时失误率降低50%

**类型3: 拦截者 (Interceptor)**
- 高拦截能力，擅长阻断敌方能量球
- 技能: "精准拦截" - 拦截范围+1

**类型4: 干扰者 (Disruptor)**
- 可降低附近敌方单位的能力值
- 技能: "电磁干扰" - 相邻排敌方单位发送能力-20%

**类型5: 储能者 (Capacitor)**
- 可以储存能量球并在后续回合释放
- 技能: "能量储备" - 最多储存2个能量球

### 3.3 单位获取

- 游戏开始前，玩家构建自己的牌组（11张单位卡）
- 每轮布阵阶段，从牌组中选择单位放置到战场
- 单位被击败后进入墓地，本轮无法再次使用

## 4. 卡牌系统

### 4.1 单位卡

- 代表可部署的单位
- 包含单位的所有属性信息
- 部分单位卡有使用限制（如每轮最多使用X次）

### 4.2 技能卡 (可选扩展)

- 一次性使用的特殊效果
- 例如：临时提升某排单位能力、立即发射额外能量球等

### 4.3 装备卡 (可选扩展)

- 可装备给单位的增强道具
- 例如：提升发送能力的装备、增加拦截范围的装备等

## 5. 游戏流程详解

### 5.1 游戏开始

1. 玩家选择牌组
2. 系统随机决定先手方
3. 双方大本营初始生命值：100点

### 5.2 单轮流程

```
┌─────────────────────────────────────────────────────┐
│                    第N轮开始                         │
├─────────────────────────────────────────────────────┤
│  1. 布阵阶段 (进攻方)                                │
│     - 查看当前手牌                                   │
│     - 调整每排单位配置                               │
│     - 确认布阵                                       │
├─────────────────────────────────────────────────────┤
│  2. 进攻阶段                                         │
│     a. A1决定发射能量球数量                          │
│     b. 对每个能量球执行传递链:                       │
│        A1 → A2 → A3 → A4 → 敌方大本营               │
│        - 每步判断: 成功/失误/拦截/转移              │
│     c. 结算伤害                                      │
├─────────────────────────────────────────────────────┤
│  3. 角色互换，重复步骤1-2                            │
├─────────────────────────────────────────────────────┤
│  4. 轮次结束，检查胜利条件                           │
└─────────────────────────────────────────────────────┘
```

### 5.3 能量球传递判定流程

```
能量球从单位S发送到单位T:

1. 目标选择
   └─> S根据感知范围和策略选择目标

2. 失误判定
   └─> 计算失误概率
       └─> 失败? 能量球丢失，流程结束
       └─> 成功? 继续

3. 拦截判定  
   └─> 找出路径上最近的敌方单位E
   └─> 计算E的拦截成功率
       └─> 成功? 能量球被拦截，流程结束
       └─> 失败? 继续

4. 到达判定
   └─> 能量球到达目标排
   └─> 选择接收单位
       └─> 同排转移?
           └─> 是: 标记"已转移"，新单位作为S，回到步骤1
           └─> 否: 能量球进入该单位

5. 链条继续
   └─> 如果当前是A4，则对敌方大本营造成伤害
   └─> 否则，该单位成为新的发送方，继续传递
```

## 6. 数值系统

### 6.1 基础公式

**发送失误率**:
```
失误率 = 基础失误率(30%) 
       × (1 - 发送方发送能力/100) 
       × (1 - 接收方接收能力/100)  
       × (1 + 距离/10)
最小值: 5%, 最大值: 90%
```

**拦截成功率**:
```
拦截率 = 基础拦截率(40%)
       × (拦截者拦截能力/100)
       × (1 - 距离到路径/5)
最小值: 5%, 最大值: 95%
```

**伤害计算**:
```
伤害 = 到达大本营的能量球数量 × 单球伤害(10点)
```

### 6.2 平衡性考虑

- 每轮平均伤害应控制在30-40点
- 3轮内应有足够的策略深度
- 落后方应有翻盘机会
